Notes

Demo 1 — A web server is required when developing with the JavaScript API. Launching an application in your browser using the file protocol (file://) can result in runtime errors. Error message indicates an issue in the JS API code.

Demo 2 — The esri.css file is a required application component. Failing to include it doesn't result in an application error, but does result in faulty execution.

Building applications which include Dojo functionality in the HTML also requires adding one of Dojo's CSS stylesheets and setting the body tag's class attribute to the Dojo class. The code will execute without error, but the application will not include the requested Dojo functionality.

Demo 3 — Setting the language attribute in the HTML tag (<html lang="en">) results in an error in the JS API. The page loads but not in the expected way.

Demo 4 — Trailing comma at the end of the require alias list results in a syntax error. Error reported in the console.

Demo 5 — Missing module in require results in a reference error, constructor class not defined.

Demo 6 — Modules not listed in the same order can generate unusual results.

Demo 7 — Failing to run the Dojo parser when declaratively implementing dijits results in unexpected results. No error in the console.

Demo 8 — Creates a dijit tabbed UI and loads maps into separate tabs. First map fails to load, but second and third do. No errors in the console.

Demo 9 — Layers are not added to the map when failing to provide the appropriate parameter to the addLayers method. No error in the console.

Demo 10 — Legend is added to the application but doesn't appear. JavaScript is case sensitive. Legend doesn't load when byId method is misspelled. No error in the console.
